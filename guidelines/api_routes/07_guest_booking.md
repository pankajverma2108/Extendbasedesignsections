# Guest Booking API Routes

> Base URL: `http://localhost:8080` (dev) or Railway URL (prod)

---

## 1. GET `/guest/booking/rooms`

Browse available room types for a property + date range. **No auth required** — visitors can browse.

**Query Params**:

| Param | Type | Required | Example |
|---|---|---|---|
| property_id | string | Yes | prop-bandra-001 |
| checkin | date string | Yes | 2026-04-01 |
| checkout | date string | Yes | 2026-04-03 |

**Response** (200):
```json
{
  "property_id": "prop-bandra-001",
  "checkin_date": "2026-04-01",
  "checkout_date": "2026-04-03",
  "no_of_nights": 2,

    {
      "id": "rt-6dorm",
```

## 2. POST `/guest/booking/create-order`

Validates the full booking cart (rooms + optional addons), reserves inventory, creates pending booking records. Returns summary for payment.

**Auth**: Guest JWT required

**Request**:
```json
{
  "property_id": "prop-bandra-001",
  "checkin_date": "2026-04-01",
  "checkout_date": "2026-04-03",
  "rooms": [
    { "room_type_id": "rt-6dorm", "quantity": 2 },
    { "room_type_id": "rt-queen", "quantity": 1 }
  ],
  "addons": [
    { "product_id": "prod-toilet-kit", "quantity": 2 },
    { "product_id": "prod-water-bottle", "quantity": 3 }
  ]
}
```

**Response** (201):
```json
{
  "ezee_reservation_id": "VH-BANDRA-MN021NE0-35A1",
  "property_id": "prop-bandra-001",
  "property_name": "Vibe House Bandra",
  "checkin_date": "2026-04-01",
  "checkout_date": "2026-04-03",
  "no_of_nights": 2,
  "total_guests": 3,
  "rooms": [
    {
      "room_type_id": "rt-6dorm",
      "room_type_name": "6 Bed Mixed Dormitory",
      "type": "DORM",
      "quantity": 2,
      "price_per_night": 449,
      "line_total": 1796
    },
    {
      "room_type_id": "rt-queen",
      "room_type_name": "Queen Size Room",
      "type": "PRIVATE",
      "quantity": 1,
      "price_per_night": 1999,
      "line_total": 3998
    }
  ],
  "addons": [
    {
      "product_id": "prod-toilet-kit",
      "product_name": "Toilet Kit",
      "quantity": 2,
      "unit_price": 150,
      "line_total": 300
    }
  ],
  "subtotal_rooms": 5794,
  "subtotal_addons": 600,
  "grand_total": 6394,
  "addon_order_id": "f852744f-...",
  "status": "PENDING_PAYMENT"
}
```

**What happens**:
1. Validates room availability against overlapping bookings
2. Validates addon stock for COMMODITY items
3. Reserves addon inventory (`available_stock--`, `reserved_stock++`)
4. Creates `ezee_booking_cache` (status: PENDING_PAYMENT)
5. Creates `booking_guest_access` (role: PRIMARY)
6. Creates `addon_orders` + `addon_order_items` (if addons)
7. Creates `booking_slots` for each guest

**Errors**:
- `400` — No rooms selected, insufficient stock, borrowable/free items in cart
- `404` — Property or room type not found

## 3. POST `/payment/create-booking-order`

Creates a Razorpay order for the pending booking. Called after `create-order` returns the summary.

**Auth**: Guest JWT required

**Request**:
```json
{
  "ezee_reservation_id": "VH-BANDRA-MN021NE0-35A1",
  "grand_total": 6394,
  "addon_order_id": "f852744f-..."
}
```

**Response** (201):
```json
{
  "razorpay_order_id": "order_STo4oZS8o4gD9C",
  "razorpay_key": "rzp_test_...",
  "amount": 6394,
  "amount_paise": 639400,
  "currency": "INR",
  "payment_id": "82700941-...",
  "ezee_reservation_id": "VH-BANDRA-MN021NE0-35A1",
  "guest": { "email": "samir@gmail.com" }
}
```

## Payment After Booking

### On Success (verify or webhook)
- Booking status: `PENDING_PAYMENT` → `CONFIRMED`
- Addon inventory: `reserved_stock--`, `sold_count++`
- Addon order: `PENDING` → `PAID`
- Payment: `CREATED` → `CAPTURED`

### On Failure
- Booking status: `PENDING_PAYMENT` → `CANCELLED` (is_active=false)
- Addon inventory: `available_stock++`, `reserved_stock--` (released back)
- Addon order + items: deleted
- Booking slots + guest access: deleted
- Payment: `CREATED` → `FAILED`

---

## Booking Status Lifecycle

```
PENDING_PAYMENT  →  CONFIRMED     (payment captured)
PENDING_PAYMENT  →  CANCELLED     (payment failed / expired)
CONFIRMED        →  PENDING_SYNC  (future: queued for eZee sync)
PENDING_SYNC     →  SYNCED        (future: pushed to eZee)
```

---

## ERI Format

Local ERIs generated by Vibe House follow this pattern:
```
VH-{PROPERTY_CODE}-{TIMESTAMP_BASE36}-{RANDOM_4}
Example: VH-BANDRA-MN021NE0-35A1
```

When eZee integration is live, the Kafka sync worker will push the booking to eZee and update/map the ERI.

## Testing in Postman (Step by Step)

### Step 1: Browse rooms (no auth)
```
GET /guest/booking/rooms?property_id=prop-bandra-001&checkin=2026-04-01&checkout=2026-04-03
→ see available room types + pricing
```

### Step 2: Login as guest
```
POST /guest/auth/login
{ "email": "samir@gmail.com", "password": "Vibe@2026!" }
→ copy access_token
```

### Step 3: Create booking order
```
POST /guest/booking/create-order
Authorization: Bearer <token>
{
  "property_id": "prop-bandra-001",
  "checkin_date": "2026-04-01",
  "checkout_date": "2026-04-03",
  "rooms": [{"room_type_id": "rt-6dorm", "quantity": 2}],
  "addons": [{"product_id": "prod-toilet-kit", "quantity": 1}]
}
→ get ezee_reservation_id, grand_total, addon_order_id
```

### Step 4: Create Razorpay payment
```
POST /payment/create-booking-order
Authorization: Bearer <token>
{
  "ezee_reservation_id": "<eri>",
  "grand_total": <total>,
  "addon_order_id": "<id or omit>"
}
→ get razorpay_order_id
```

### Step 5: Simulate payment (dev only)
```
POST /payment/dev/simulate-capture
{ "razorpay_order_id": "<rzp_order_id>" }
→ "Booking confirmed, payment captured"
```

### Step 5b: Or simulate failure
```
POST /payment/dev/simulate-fail
{ "razorpay_order_id": "<rzp_order_id>" }
→ "Booking payment failed. Pending booking rolled back."
```

## Inventory Reservation Flow

```
                     create-order              payment captured
                    ┌────────────┐            ┌────────────────┐
available_stock ────┤ decrement  ├────────────┤  (no change)   │
                    └────────────┘            └────────────────┘
                    ┌────────────┐            ┌────────────────┐
reserved_stock  ────┤ increment  ├────────────┤  decrement     │
                    └────────────┘            └────────────────┘
                                              ┌────────────────┐
sold_count      ──────────────────────────────┤  increment     │
                                              └────────────────┘

                     payment failed
                    ┌────────────────┐
available_stock ────┤  increment     │  (restored)
                    └────────────────┘
                    ┌────────────────┐
reserved_stock  ────┤  decrement     │  (released)
                    └────────────────┘
```
